# SKILL — SQR Vendas (Sistema de Qualificação e Relacionamento)

## Visão Geral
Ferramenta pessoal de vendas com SPIN Selling automatizado via WhatsApp.
O usuário cadastra leads → o sistema conduz o ciclo de vendas completo (disparo + resposta automática) via Evolution API + Claude API.

**Produtos alvo:**
- Solara Connect (clínicas: médica, odontológica, estética, fisioterapia, harmonização, emagrecimento)
- Autoracer (marketplace de veículos — Vale do Paraíba)

## Banco de Dados — Supabase

### Tabela: `leads`
```sql
create table leads (
  id uuid primary key default gen_random_uuid(),
  nome text not null,
  telefone text not null unique,  -- formato: 5512999999999
  produto text not null,          -- 'solara_connect' | 'autoracer'
  segmento text,                  -- ex: 'clinica_estetica', 'revendedora'
  cidade text,
  observacoes text,
  estagio text default 'novo',    -- novo | situacao | problema | implicacao | need | proposta | fechado | perdido
  ativo boolean default true,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

### Tabela: `mensagens`
```sql
create table mensagens (
  id uuid primary key default gen_random_uuid(),
  lead_id uuid references leads(id),
  direcao text not null,          -- 'enviada' | 'recebida'
  conteudo text not null,
  estagio_spin text,              -- estágio no momento da mensagem
  created_at timestamptz default now()
);
```

### Tabela: `campanhas`
```sql
create table campanhas (
  id uuid primary key default gen_random_uuid(),
  nome text not null,
  produto text not null,
  status text default 'rascunho',  -- rascunho | ativa | pausada | concluida
  total_leads int default 0,
  disparados int default 0,
  responderam int default 0,
  created_at timestamptz default now()
);
```

## Lógica SPIN por Produto

### Estágios e objetivos

| Estágio | Objetivo | Gatilho para avançar |
|---|---|---|
| `novo` | Lead cadastrado, aguardando disparo | Disparo manual ou campanha |
| `situacao` | Entender o contexto atual | Lead responde sobre sua operação |
| `problema` | Identificar a dor real | Lead menciona dificuldade/frustração |
| `implicacao` | Aprofundar consequências do problema | Lead reconhece impacto do problema |
| `need` | Lead verbaliza o valor da solução | Lead demonstra interesse/pergunta sobre solução |
| `proposta` | Enviar proposta/demo | Após need confirmado |
| `fechado` | Venda concluída | Confirmação |
| `perdido` | Sem interesse | Recusa explícita ou 3+ sem resposta |

### Prompts SPIN — Solara Connect (Clínicas)

**Sistema base:**
```
Você é um consultor de vendas especializado em automação para clínicas de saúde e estética no Brasil.
Produto: Solara Connect — agente de atendimento com WhatsApp integrado, agendamento, prontuário, recuperação de clientes e relatórios.
Tom: profissional, próximo, sem pressão. Nunca force a venda. Conduza com perguntas.
Responda SEMPRE em português brasileiro informal mas profissional.
Máximo 3 frases por mensagem. Seja direto e humano.
```

**Mensagem inicial (estágio: situacao):**
```
Prompt: Gere a primeira mensagem para {nome}, {segmento} em {cidade}.
Objetivo: apresentação breve + pergunta sobre como gerenciam o atendimento hoje.
Não mencione o produto ainda. Apenas abra a conversa com curiosidade genuína.
```

**Estágio problema:**
```
Prompt: O lead {nome} respondeu: "{ultima_mensagem}".
Estágio atual: problema.
Objetivo: identificar a principal dor (agenda cheia/vazia, no-show, pacientes sumindo, WhatsApp manual caótico).
Faça UMA pergunta que aprofunde o problema mencionado. Não ofereça solução ainda.
```

**Estágio implicação:**
```
Prompt: O lead {nome} disse: "{ultima_mensagem}".
Estágio atual: implicacao.
Objetivo: fazer o lead perceber o custo real do problema (tempo perdido, receita não gerada, desgaste da equipe).
Use linguagem de impacto financeiro ou operacional. UMA pergunta de implicação.
```

**Estágio need (need-payoff):**
```
Prompt: O lead {nome} disse: "{ultima_mensagem}".
Estágio atual: need.
Objetivo: fazer o lead imaginar como seria se o problema estivesse resolvido.
Pergunte sobre o cenário ideal. Ex: "Se você pudesse recuperar automaticamente os pacientes que sumiam, como isso impactaria sua clínica?"
```

**Estágio proposta:**
```
Prompt: O lead {nome} demonstrou interesse. Última mensagem: "{ultima_mensagem}".
Apresente o Solara Connect em 3 bullets objetivos alinhados exatamente com as dores mencionadas ao longo da conversa.
Finalize com CTA para uma demonstração rápida (15min). Sem pressão.
Histórico de dores identificadas: {historico_dores}
```

### Prompts SPIN — Autoracer (Veículos)

**Sistema base:**
```
Você é um consultor de vendas especializado em marketplaces automotivos para o Vale do Paraíba, SP.
Produto: Autoracer — marketplace de veículos com sistema de vídeo curto, para revendedoras e vendedores da região.
Tom: direto, regional, sem enrolação. Fale como alguém da região.
Máximo 3 frases. Português brasileiro informal.
```

**Mensagem inicial:**
```
Prompt: Gere a primeira mensagem para {nome}, {segmento} em {cidade}.
Objetivo: abertura descontraída + pergunta sobre como anunciam veículos hoje (Instagram? OLX? boca a boca?).
```

**Estágio problema:**
```
Prompt: O lead {nome} respondeu: "{ultima_mensagem}".
Identifique a dor: pouco alcance local, custo alto de plataformas grandes (iCarros, Webmotors), dificuldade de destacar o estoque, sem vídeo.
UMA pergunta que aprofunde. Sem solução ainda.
```

**Estágio implicação:**
```
Prompt: Lead disse: "{ultima_mensagem}".
Faça o lead calcular mentalmente o custo de não ter uma vitrine local forte. Quantos veículos ficam parados? Quanto representa em comissão perdida?
```

**Estágio need:**
```
Prompt: Lead disse: "{ultima_mensagem}".
Faça o lead imaginar ter uma plataforma local onde compradores do Vale do Paraíba buscam especificamente. Vídeo curto do veículo em destaque.
```

**Estágio proposta:**
```
Prompt: Apresente o Autoracer alinhado às dores de {nome}.
Destaque: alcance local no Vale do Paraíba, vídeo curto do estoque, custo acessível vs grandes plataformas.
CTA: convide para ver a plataforma funcionando. Sem pressão.
Dores identificadas: {historico_dores}
```

## Backend — Endpoints FastAPI

```python
# routers/webhook.py
POST /webhook/whatsapp
# Recebe mensagem do Evolution API
# → identifica lead pelo telefone
# → registra mensagem no Supabase
# → chama claude_spin.py com contexto completo
# → envia resposta via Evolution API
# → atualiza estágio do lead

# routers/leads.py
GET    /leads               # Lista todos os leads
POST   /leads               # Cadastra novo lead
GET    /leads/{id}          # Detalhe + histórico
PUT    /leads/{id}          # Atualiza lead/estágio
DELETE /leads/{id}          # Remove lead

# routers/campanhas.py
POST /campanhas/disparar    # Dispara mensagem inicial para lista de leads
# → itera leads com estagio='novo'
# → gera mensagem inicial via Claude
# → envia via Evolution API
# → atualiza estagio para 'situacao'
```

## Regras de Negócio

1. **Uma conversa por vez por lead** — enquanto aguarda resposta, não dispara nova mensagem
2. **Máximo 3 tentativas sem resposta** → marca como `perdido` automaticamente
3. **Intervalo mínimo entre mensagens:** 24h (evitar bloqueio WA)
4. **Nunca mencionar o produto na primeira mensagem** (SPIN puro)
5. **Estágio só avança, nunca volta** (exceto override manual)
6. **Histórico de dores** é acumulativo — Claude sempre recebe o contexto completo
